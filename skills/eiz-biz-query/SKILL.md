# EIZ 业务数据查询 Skill

收到用户关于业务数据的查询需求时（如查客户、查订单、查物流等），按以下步骤执行：

## 第一步：从知识库获取规范

使用**知识库检索工具**（通常名为 `knowledge_mcp_server-Retrieve_Documents` 或类似名称，根据当前环境的实际可用工具名使用），按关键词查询相关内容，至少覆盖：

1. **查询规范** — 输入关键词如 `MySQL 查询规范 索引`，获取：
   - 哪些表是大表、必须带 WHERE + 索引字段
   - 明细查询必须带 LIMIT
   - 大表禁止 JOIN，必须分步查询
   - 时间范围查询禁止用函数包裹字段
   - 只读库，禁止任何写操作

2. **表结构与字段说明** — 输入关键词如 `表字段说明 索引`，获取：
   - 涉及的表有哪些字段、字段含义
   - 哪些字段有索引，WHERE 条件该用什么

3. **查询模板** — 输入关键词如 `查询模板 SQL`，获取：
   - 该类型查询的标准写法
   - 状态码翻译规则
   - 敏感字段脱敏方式

## 第二步：确认表结构（首次或不确定时）

使用**数据库查询工具**（通常名为 `mysql-search_objects` 和 `mysql-execute_sql` 或类似名称，根据当前环境的实际可用工具名使用）执行：

```sql
DESCRIBE 表名;
SHOW INDEX FROM 表名;
```

确认字段名和索引后再写正式查询。

## 第三步：编写并执行查询

根据知识库规范编写 SQL，通过**数据库执行工具**（如 `mysql-execute_sql`）执行。关键要求：

- **所有查询必须只读**（SELECT / SHOW / DESCRIBE）
- **WHERE 条件命中索引字段**，不确定时先 SHOW INDEX
- **明细查询带 LIMIT**（默认不超过 50 条）
- **大表不 JOIN**，拆成多步分步查询
- **时间范围用 `>=` 和 `<`**，不用 MONTH() / DATE() 包裹字段
- **结果展示**：状态码翻译成中文，金额保留 2 位小数，时间格式化为 YYYY-MM-DD HH:mm
- **敏感信息脱敏**：email、phone 等字段用 CONCAT 做部分遮掩

## 第四步：呈现结果

用清晰易懂的中文向用户解释查询结果，必要时补充业务背景。

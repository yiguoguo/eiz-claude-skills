# 女娲 · Skill 造人术

> "你写不进 SKILL.md 的部分，才是你真正的护城河。" 但能写进来的部分，已经足够强大。

女娲不复制人——它提取思维框架。一个好的 perspective skill 是一个可运行的认知操作系统。

## 核心理念

一个 perspective skill 需要回答：

- 这个人用什么心智模型看世界？（**lenses**）
- 做判断时用什么决策启发式？（**rules of thumb**）
- 怎么表达自己？（**DNA**）
- 绝对不做什么？（**anti-patterns**）
- 这个 skill 做不到什么？（**honesty boundaries**）

**关键区分：捕捉他们怎么思考，而不是他们说了什么。**

## 使用方式

```
/nuwa 张一鸣
/nuwa 查理芒格 --focus 投资决策
/nuwa 价值投资 --type topic
/nuwa 张一鸣 --update zhangyiming-perspective
```

## 执行流程

### Phase 0：确认需求（30 秒）

收到输入后，确认：
1. **人物/话题**：确保理解正确
2. **方向**（可选）：全景画像 vs 聚焦某个维度？
3. **用途**：思维顾问？决策参考？角色扮演？
4. **新建还是更新**：是否已有该人物的 skill？

如果用户只说"做 XX"没有额外说明 → 默认全景画像 + 思维顾问，直接执行。

### Phase 0.5：创建 Skill 目录

确认后立即创建目录结构：

```
[perspectives/]person-name-perspective/
├── SKILL.md                          # 最终交付物
└── references/
    ├── research/                     # 各 Agent 研究产出（必须保存）
    │   ├── 01-writings.md            # 著作与系统性思考
    │   ├── 02-conversations.md       # 长对话与即兴思考
    │   ├── 03-expression-dna.md      # 碎片化表达与风格 DNA
    │   ├── 04-external-views.md      # 外部视角与批评
    │   ├── 05-decisions.md           # 决策记录与行为
    │   └── 06-timeline.md            # 人物时间线
    └── sources/                      # 下载的原始材料
        ├── books/
        ├── transcripts/
        └── articles/
```

**硬性规则：每个子 Agent 必须将研究结果保存到对应的 md 文件。研究不落盘 = 研究不存在。** Skill 必须自包含——复制整个 skill 目录即可独立运行。

### Phase 1：多源研究（并行 Agent 集群）

启动 6 个并行 Agent，各负责一个信息维度：

#### Agent 1：著作研究

- **搜索目标**：书籍、文章、论文、newsletter
- **提取重点**：核心论点（重复 ≥3 次 = 真实信念）、自创术语、推荐书单
- **输出文件**：`01-writings.md`

#### Agent 2：对话研究

- **搜索目标**：播客、长视频、AMA、深度访谈
- **提取重点**：压力下的回应、即兴类比、立场变化、拒绝回答的问题
- **输出文件**：`02-conversations.md`

#### Agent 3：表达研究

- **搜索目标**：Twitter/X、微博、短帖
- **提取重点**：高频句式、争议立场、幽默风格、公开论战
- **输出文件**：`03-expression-dna.md`

#### Agent 4：外部视角

- **搜索目标**：他人的分析、书评、批评、传记
- **提取重点**：外部观察到的模式、批评与争议、同行比较
- **输出文件**：`04-external-views.md`

#### Agent 5：决策研究

- **搜索目标**：重大决策、转折点、争议行为
- **提取重点**：决策背景与逻辑、事后反思、言行一致性
- **输出文件**：`05-decisions.md`

#### Agent 6：时间线

- **搜索目标**：出生/出道至今
- **提取重点**：关键里程碑、思想转折点、最近 12 个月（防止信息过时）
- **输出文件**：`06-timeline.md`

#### 每个 Agent 的硬性要求

- 研究结果**必须保存**到 `references/research/0X-xxx.md`
- 注明来源和可信度（一手 > 二手 > 推断）
- 区分"他们说的" vs "别人说他们" vs "我推断的"
- 发现矛盾时保留，不要抹平

#### 信息来源优先级

| 来源类型 | 揭示内容 | 权重 |
|---|---|---|
| 本人著作 | 系统性思考 | 最高 |
| 长对话/访谈 | 即兴思维过程 | 最高 |
| 实际决策记录 | 真实行为 vs 表态 | 最高 |
| 社交媒体 | 表达风格、即时反应 | 中等 |
| 他人评价 | 外部视角、盲点 | 中等 |
| 二手转述 | 参考但需验证 | 低 |

#### 来源黑名单

中文人物排除：知乎、微信公众号、百度百科/知道。只接受权威中文媒体：36Kr、极客公园、晚点、财新、第一财经、虎嗅、少数派等。

#### Agent 失败处理

- 超时/无结果：不等待，继续流程。Phase 2 中标记"信息不足"
- 来源稀缺（<10 个可用来源）：Phase 0 提示用户，降低预期，扩大诚实边界
- Agent 结果冲突：保留矛盾——矛盾是有价值的信号

**核心原则：诚实的 60 分 skill（明确标注局限）优于看似完美的 90 分 skill（掺杂编造信息）。**

### Phase 1.5：研究质量检查点

所有 Agent 完成后暂停，展示研究质量摘要：

```
┌──────────────────┬──────────┬──────────────────────────┐
│ Agent            │ 来源数   │ 关键发现                 │
├──────────────────┼──────────┼──────────────────────────┤
│ 1 著作           │ 8        │ 核心论点：...            │
│ 2 对话           │ 5        │ 立场变化：...            │
│ 3 表达           │ 120      │ 高频词："..."            │
│ 4 外部           │ 6        │ 主要批评：...            │
│ 5 决策           │ 4        │ 关键决策：...            │
│ 6 时间线         │ 完整     │ 最新：2026年3月...       │
├──────────────────┼──────────┼──────────────────────────┤
│ 矛盾             │ 2        │ Agent1 说 X，Agent4 说 Y │
│ 信息缺口         │ 无       │                          │
└──────────────────┴──────────┴──────────────────────────┘
```

用户确认质量 → 进入 Phase 2。用户想补充某个维度 → 补充后再继续。

### Phase 2：框架提取（综合分析）

#### 2.1 心智模型提取（3-7 个）

识别标准：
- **跨领域复现**：同一框架出现在 2+ 不同领域 → 真实心智模型
- **有自创术语**：给这个思维模式命名了 → 核心信念
- **有预测力**：能预测他们对新问题的立场

每个模型记录：名称、一句话描述、来源证据（≥2 个场景）、应用方法、局限性

#### 2.2 决策启发式提取（5-10 个）

= 这个人做判断时的快速规则。能表达为"如果 X，就 Y"，有具体案例支撑。

#### 2.3 表达 DNA 分析

| 维度 | 提取内容 |
|---|---|
| 句式偏好 | 长句/短句、疑问/陈述、类比密度 |
| 用词特征 | 高频词、自创词、禁忌词 |
| 节奏 | 结论先行还是铺垫先行、转折风格 |
| 幽默风格 | 讽刺/自嘲/荒诞/冷幽默/无 |
| 确定性表达 | "我不确定"型 vs "显然"型 |
| 引用习惯 | 引用谁、引用什么类型 |

#### 2.4 价值观与反模式

- **核心价值观**：3-5 个，按优先级排序
- **反模式**：这个人明确反对的行为/思维方式
- **矛盾与张力**：价值观之间的内部冲突（这是深度的来源）

#### 2.5 知识谱系

谁影响了他们 → 他们 → 他们影响了谁 → 在知识地图上的位置

#### 2.6 诚实边界

必须明确声明的局限：
- 无法预测他们对全新问题的反应
- 无法替代这个人的创造力和直觉
- 公开表达 vs 私下想法可能不同
- 研究截止日期的信息限制

### Phase 3：Skill 构建

基于 Phase 2 的结果组装 SKILL.md，结构如下：

```markdown
---
name: [person-name]-perspective
description: "[人物名]的思维方式。[一句话总结其核心思维特征]。"
disable-model-invocation: true
---

# [人物名] Perspective

[一段话概述这个人是谁、为什么值得提取其思维框架]

## 核心心智模型

### 1. [模型名称]
[一句话描述]

**来源证据：**
- [场景1]
- [场景2]

**应用方法：** [什么时候用这个模型]
**局限：** [这个模型在什么情况下失效]

### 2. ...

## 决策启发式

- [规则1]：[解释]（案例：...）
- [规则2]：...

## 表达 DNA

[按 2.3 的维度描述表达特征]

## 价值观与反模式

**核心价值观：**
1. ...
2. ...

**反模式：**
- ...

## 使用指南

当你遇到 [XX 类型] 问题时，用这个 skill 的方式思考：
1. ...
2. ...

## 诚实边界

这个 skill 无法做到：
- ...
```

### Phase 4：质量验证

生成 skill 后，用 3 个独立测试验证：

#### 4.1 已知测试（Sanity Check）

选 3 个该人物公开回答过的问题，用新 skill 作答，与实际立场对比：
- 方向一致 → 模型有效
- 偏差 → 回溯调整心智模型权重

#### 4.2 边界测试

选 1 个他们没公开讨论过但相关的问题，用 skill 推理：
- 预期输出："基于模型 X 和 Y，可能倾向于...但不确定"
- 不应该给出绝对确定的答案

#### 4.3 声音测试

用 skill 写一段 100 字的分析，判断：
- 有这个人的表达特征吗？
- 不是泛泛的 AI 套话？
- 不是原话拼凑？

#### 4.4 通过标准

| 检查项 | 通过 | 失败信号 |
|---|---|---|
| 心智模型数量 | 3-7 个，每个有来源证据 | <3 或 >10 |
| 每个模型的局限 | 明确说明失效条件 | 只列优点 |
| 表达 DNA 辨识度 | 100 字内能认出是谁 | 读起来像通用 ChatGPT |
| 诚实边界 | ≥3 个具体局限 | 只有"无法替代本人" |
| 内部张力 | ≥2 对矛盾 | 观点高度一致（太假） |
| 一手来源比例 | >50% | 主要来自二手 |

通过 → 交付。失败 → 标记薄弱点，回 Phase 2 迭代。

将验证结果展示给用户确认后再完成。

## 更新已有 Skill

当用户说"更新 XX 的 skill"或"XX 有新动态"：
1. 读取已有 SKILL.md，记录上次研究截止日期
2. 只启动 Agent 2（最新对话）+ Agent 5（最新决策）+ Agent 6（时间线更新）
3. 对比新旧内容——强化、更新或补充
4. 增量更新 SKILL.md，不要全量重写

## 特殊场景

**在世人物 vs 历史人物**
- 在世：关注时效性，标注截止日期，建议定期更新
- 历史：材料更稳定但可能存在传记偏见，交叉验证

**话题 Skill vs 人物 Skill**
- 输入是话题（如"价值投资"）而非人物时：
  - Phase 1 搜索该话题的核心实践者和理论家
  - Phase 2 提取领域共识 + 学派分歧
  - 输出以话题为中心，引用多人视角

## 品味指南

### 好 Skill vs 坏 Skill

| 好 Skill | 坏 Skill |
|---|---|
| 能处理新问题 | 只能复述旧语录 |
| 有内部矛盾 | 观点高度一致（假） |
| 承认不确定性 | 什么都能答 |
| 3-7 个核心模型，深而锐利 | 20 条模糊"原则" |
| 表达有辨识度 | 读起来像 ChatGPT |
| 明确声明局限 | 暗示能替代本人 |

### 提取品味

- **少即是多**：3 个深邃的心智模型 > 15 条浅层原则
- **矛盾是特性，不是 Bug**：保留内部张力
- **不要美化**：盲点和局限是人格的一部分
- **可证伪**：每个心智模型都应该能被具体案例验证或推翻
- **绝不编造**：信息不足时降低预期，不捏造

### 来源品味

- 一手 > 二手：本人的话 > 别人的转述
- 长文 > 语录：一篇 3000 字的文章比 50 条推文更能揭示思维结构
- 争议 > 共识：最受争议的观点揭示最多独特性
- 变化 > 不变：立场变化比始终如一携带更多信息
- 行为 > 表态：实际决策 > 公开声明
